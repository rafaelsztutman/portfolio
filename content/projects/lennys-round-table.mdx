## What it does

The round table is the advisory board you wish you had. You drag three guests from a curated 287-person roster — Marty Cagan, Keith Rabois, Elena Verna, April Dunford, whoever — onto seats at a felt table, pose a challenge (pricing, hiring, launching, AI agent design, anything operator-shaped), and the seated guests respond *as themselves*. Every response is grounded in stance cards extracted from the guest's real episodes. Tier-1 responses include a verbatim quote with the timestamp it came from. Multi-turn back-and-forth — Casey Winters can directly push back on Jason Lemkin. When three seats are filled, a "Deal a hand" button appears in the center of the felt and you can play no-limit Hold'em with the same three guests, each with persona-driven betting behaviour. There's a public demo mode that replays curated conversations turn-by-turn without burning LLM credits, and an invite-token path that unlocks the live experience.

## Why I built it

Lenny ran a buildathon on his podcast + newsletter archive — a 15-day window, judged by Lenny and the Replit team. The shelf was already crowded: LennyRPG, Tiny Stakeholders, Lenny Playbook, Lenny Listens, Advice Arena, half a dozen more. I wanted an angle that wasn't another search box, another database, or another whimsy app. The thing I kept coming back to: I read the newsletter and listen to the podcast specifically because I want to ask the guests questions about my own work. I can't, so I imagine what they'd say. What if I could just… ask them? The product hypothesis: a *small* advisory dynamic — three guests, real disagreements, real citations — beats any chat-with-the-whole-archive feature on something that's actually useful to operators.

## How it works

Each guest is an offline-baked persona pack — a `~50-80 KB` JSON file produced by a 9-stage Node pipeline. Stages: preprocess (filter the guest's speech turns out of speaker-tagged transcripts) → bio → voice anchors (5–10 verbatim quotes for tone imitation) → stance cards (the core unit: ~30–80 "this guest believes X" claims per guest, each with a verbatim quote, citation, confidence score, and topic tag, **programmatically validated against the transcript** — if the substring isn't a contiguous match, the stance is dropped) → frameworks → adjacent topics → chunk embeddings → disagreement graph (the model finds cross-guest disagreements by topic, weighted toward known-contrarian voices; Keith Rabois alone has 19) → assemble. End state: 287 personas, 3,644 verbatim-validated quotes, 82 cross-guest disagreements, all committed as plain JSON.

A single user message goes through three server phases. **Intent scoring** runs a small Claude model in parallel per seated guest — each returns `want_to_speak`, a one-line intent, and which stance cards they'd lean on. A threshold filter picks who actually responds; a guest with nothing useful to add stays silent. **Tiered response** runs a larger model for the selected guests; the prompt includes the guest's persona pack, the other seated guests' names so they can address each other, and the disagreement graph entries that intersect this guest with the others at the table — surfaced as a `WHERE YOU MATERIALLY DIFFER FROM OTHERS` block. Responses are constrained to a 5-tier hierarchy: tier 1 must include a verbatim quote; tier 4 is free synthesis with no quote; tier 5 is "I have nothing useful here." Quote-bearing responses are **runtime-validated** against the persona pack's quote pool — if the model invents a quote, the response auto-downgrades to tier 4 instead of publishing the fake. **Follow-up beats** re-run intent with new context, bounded, letting guests react to each other's takes.

The roster sidebar is hybrid search: substring filter on names + a server-side keyword recommender (PM-specific synonym map, ~10 ms, $0 per query) + a client-side semantic search (Xenova/all-MiniLM-L6-v2 loaded in-browser on first focus, ~6 MB int8 weights, ~110 KB persona index). Both run in parallel on every keystroke; keyword renders first, semantic silently overwrites when it arrives. Three-mode deployment (`demo` / `production` / `dev`) gates LLM endpoints with per-IP rate limiting, same-origin checks, and strict zod input bounds — public visitors land in demo mode and see curated replays; invite tokens unlock the live experience; the owner has password access in dev mode.

## What I learned

Three things stuck.

**LLMs imitating real people will invent quotes that sound exactly right.** This is the failure mode that scared me most — not off-color responses, not confabulated facts, but the *plausible attributed quote* that sounds exactly like something the person would say, that a user might screenshot to Twitter, that ends up in someone's slide deck six months later. Solving it took longer than building the UI. The architecture that survived: extract stances offline with verbatim quotes, validate every quote as a substring against the source transcript, drop any stance whose quote doesn't validate. At runtime, classify every response into a 5-tier grounding hierarchy and **auto-downgrade to a no-quote tier if the model's quote fails validation**. Keep the answer. Drop the unverifiable quote. Never publish the fake. The product isn't shippable without that downgrade path.

**Disagreement is a property of the agents, not a feature of the UI.** I built no banner for cross-guest disagreement, no badge, no animated lightning bolt between portraits. Instead the prompt for each seated guest includes a `WHERE YOU MATERIALLY DIFFER FROM OTHERS` block — populated with auto-detected disagreement entries from a pre-built graph that intersect this guest's stances with the other seated guests'. When the friction shows up in dialogue it's because the friction is real — you can trace it back to specific things real people said on a real podcast. The lesson generalises: behaviour you want to *feel* emergent should live in the data and the prompts, not in the UI chrome. UI banners are how you signal a feature is there; absence of banners is how you signal a behaviour is real.

**The thing that makes a demo memorable is almost never on the spec.** Eight days in, mostly as a joke, I built a poker side-mode where the three seated guests play no-limit Hold'em with persona-driven betting behaviour pulled from a per-guest decision function. It is not a serious poker product. It is the 90-second emotional cold-open that makes the guests feel like *people*, not chatbots. Watching Keith Rabois shove all-in on the turn while Marty Cagan folds for the third hand in a row is the moment when the round table stops being an LLM demo and starts being a place you want to spend time in. Every single early tester clicked the poker button before asking a real question. *Every one.* The lesson: leave room in the schedule for the late-stage detour that arrives uninvited.
